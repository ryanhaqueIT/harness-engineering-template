# Phase 2: Generate

Using the profile and analysis from Phases 0-1, generate the full harness into the target repository. Every file comes from THIS bootstrap repo — copy, configure, and write.

## Idempotency Rules

Before writing ANY file in the target repo:

1. **File does not exist** → Write it.
2. **File exists and is unchanged from our previous generation** → Overwrite (update).
3. **File exists and has been modified by the user** → SKIP and report "skipped (user-modified)".
4. **Documentation files (.md)** → Merge: append new sections, never remove existing content.
5. **JSON/YAML config** → Deep merge: add new keys, preserve existing values.

To detect user modifications: check for the watermark comment at the top of generated scripts:
```
# Generated by harness-engineering-bootstrap
```
If the file exists but lacks this watermark, it was written by the user — do not overwrite.
